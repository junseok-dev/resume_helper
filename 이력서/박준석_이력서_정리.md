### 2. 쇼특허 (Short-Cut) — AI 기반 특허 침해 리스크 분석 시스템
**기간** 2026.01.30 ~ 2026.02.03  
**역할** Core Developer (핵심 로직 구현 및 시스템 최적화)  
**기술** `Python` `OpenAI API` `RAG` `Pinecone` `Streamlit` `Plotly`

- 사용자 아이디어 입력 시 유사 특허 검색 및 침해 리스크 분석 제공
- **RAG 기반 검색 파이프라인(HyDE, Multi-Query) 설계 및 구현**
- Streamlit 기반 결과 시각화 UI 구성

**프로젝트 성과**

| 항목 | 결과 |
|------|------|
| 최종 Pass Rate | 83.1% (목표 80% 달성) |
| 평가 샘플 수 | 83개 (Golden Dataset) |
| 통과 / 실패 | 69개 통과 / 14개 실패 |
| 초기 베이스라인 | 62.5% |

**개선 과정**
- 프롬프트 엔지니어링: `NEVER FABRICATE` 지시어로 환각 억제, 특허 번호 인용 강제, 정보 부족 시 `N/A` 처리
- 데이터셋 정제: 풍력발전·배터리 등 AI/NLP 무관 Outlier 17개 제거, cosine similarity 기반 필터링
- 평가 기준 현실화: Threshold `0.7 -> 0.6` 조정으로 과도한 기준 완화

**평가 지표**

| 지표 | 의미 | 임계값 |
|------|------|------|
| Faithfulness | Context 기반 답변 여부 | >= 0.6 |
| Answer Relevancy | 질문 의도 적합성 | >= 0.6 |

- 베이스라인 62.5%에서 최종 83.1%로 개선하며 목표를 달성했고, 성과의 핵심은 프롬프트 강화와 도메인 외 데이터 정제였습니다.

| test.csv | 750건 | 69 KB |

| 항목 | 결과 |

| Recall (재현율) | 0.80 |
| Precision (정밀도) | 약 97% |

| XGBoost | 0.861 | 0.915 |
| LightGBM | 0.854 | 0.905 |
| Random Forest | 0.833 | 0.898 |
| ANN | 0.744 | 0.877 |
| Decision Tree | 0.727 | 0.851 |
| SVM | 0.629 | 0.891 |
| Logistic Regression | 0.250 | 0.802 |

**핵심 인사이트**

- Optuna 하이퍼파라미터 튜닝으로 F1-Score를 0.831에서 0.88로 추가 향상
- 오탐(False Positive) 2건 수준으로 실제 이탈자를 높은 정밀도로 검출

## 💬 Self-Introduction

데이터를 단순히 분석하는 것을 넘어, 사용자에게 직관적으로 전달하는 과정에 매력을 느껴 **프론트엔드 개발자**의 길을 선택했습니다.

빅데이터통계학과에서 데이터를 다루며, 수치 자체보다 이를 어떻게 표현하느냐에 따라 정보의 가치가 달라진다는 것을 경험했습니다. 이후 SK Networks Family AI Camp 과정에서 다양한 프로젝트를 수행하며 **데이터 처리부터 시각화, UI 구현까지 전 과정**을 경험했습니다.

특히 AI 가상 인플루언서 플랫폼 'HARI' 프로젝트에서는 Django 기반 웹 애플리케이션 위에서 HTML, CSS, JavaScript를 활용해 사용자 화면과 관리자 대시보드를 설계·구현했습니다. YouTube·Instagram API를 연동하고 Chart.js로 데이터를 시각화하며, 데이터 흐름을 이해하고 이를 사용자에게 전달하는 경험을 쌓았습니다.

앞으로도 **사용자 관점에서 가치를 전달하는 프론트엔드 개발자**로 성장하고자 합니다.
